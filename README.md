# xhs-content-moderation · 小红书内容审核 Skill

小红书（RED）笔记 **发布前违规自查** Skill。对照《社区公约 2.0》、广告法绝对化用语、站外导流红线、违禁 / 敏感词、标题党、图片水印二维码，以及医美 / 金融 / 母婴等垂类特殊限制，逐条体检，输出违规风险定级与 **可直接替换的合规改写**。

> ⚠️ 内置规则库为 **2026-07** 时点公开资料整理，**不构成法律意见**；平台规则频繁调整，一切以小红书官方最新规则与实际审核结果为准。

## 能力
- 覆盖平台硬红线（涉政 / 色情 / 赌博 / 违禁品 …）+ 广告法 + 站外导流 + 垂类专项
- 三档风险定级：🔴 硬红线（可能删帖 / 封号）· 🟠 限流风险 · 🟡 优化建议
- 每条命中给出「命中原文 → 违反哪条规则 → 为什么 → **合规改写建议**」
- 规则库见 [`references/xhs-rules.md`](./references/xhs-rules.md)（11 节 + 26 条发布前自查清单，含违禁词库与广告法例外白名单）

## 非官方声明

本项目为个人创作的内容合规自查 Skill，非小红书官方工具，未获得小红书背书或授权。它用于帮助创作者遵守公开规则，不提供绕过审核、规避平台治理或保证流量推荐的方法。

## 安装
```bash
git clone https://github.com/0lidaxiang/xhs-content-moderation.git ~/.claude/skills/xhs-content-moderation
```
或把本目录整体复制到 `~/.claude/skills/`。

## 用法
在 Claude Code 里直接说：
- “跑一遍 xhs-content-moderation，看看这篇笔记会不会违规”
- “这条小红书文案发布前帮我查一下有没有违禁词 / 限流风险”

Skill 会先确认笔记内容与垂类，再逐维度体检并出报告（Markdown，或单文件 HTML）。
